ACTIVIDAD 17: EVALUACIÓN DE VULNERABILIDADES CON CVSS v3.1

Aplicación técnica del Common Vulnerability Scoring System en un caso de estudio real.

ACTIVIDAD 17

REPORTE DE EVALUACIÓN: EVALUACIÓN DE VULNERABILIDADES CON CVSS v3.1

Alumno: Alonso Castillo Omar Karim (179033)


Materia: CNO V – Seguridad Informática

Profesor: Mtro. Servando López Contreras


Universidad Politécnica de San Luis Potosí

1. Introducción

El objetivo de esta actividad es aplicar el estándar internacional **CVSS v3.1 (Common Vulnerability Scoring System)** para cuantificar de manera rigurosa y objetiva la severidad de una vulnerabilidad de seguridad detectada en un entorno web interno. Este proceso sistemático permite a las organizaciones priorizar la mitigación de riesgos basándose en datos técnicos estructurados, medibles e independientes del fabricante.

2. Escenario de Evaluación

Se presenta el siguiente caso de estudio para su correspondiente análisis, deconstrucción y ponderación técnica:

Descripción del Escenario:

Una organización detecta una vulnerabilidad en un sistema web corporativo interno que presenta de forma precisa las siguientes características operativas:

  • 1. Puede explotarse a través de la red de datos externa o interna.
  • 2. La ejecución técnica del exploit requiere una baja complejidad de ataque.
  • 3. Se necesitan obligatoriamente privilegios elevados (administrador del sistema) para detonar la vulnerabilidad.
  • 4. No se requiere interacción alguna de un usuario legítimo (víctima).
  • 5. El alcance del ataque no cambia; es decir, no permite comprometer recursos fuera de la sandbox o del dominio de la aplicación afectada.
  • 6. El impacto directo sobre las dimensiones de Confidencialidad, Integridad y Disponibilidad es calificado técnicamente como **Bajo**.

3. Desarrollo Técnico

3.1 Construcción del Vector CVSS

Basado en la deconstrucción y mapeo de las variables descritas en el escenario de evaluación, se ha generado el siguiente vector string formal. Esta cadena representa la síntesis técnica universal que describe la vulnerabilidad:

Vector CVSS v3.1 CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:L/I:L/A:L

3.2 Justificación de las Métricas

Cada una de las métricas que componen el vector base cuenta con la siguiente justificación técnica y análisis de escenario:

Métrica Base Valor Definido Sigla Vector Justificación Técnica de Campo
Vector de Ataque (AV) De Red (Network) AV:N El fallo se ubica a nivel lógico y es explotable de forma remota a través de internet o intranets sin requerir acceso físico o local.
Complejidad de Ataque (AC) Baja (Low) AC:L No existen barreras adicionales de protección de red (como firewalls dinámicos o condiciones temporales complejas) que el atacante deba sortear.
Privilegios Requeridos (PR) Altos (High) PR:H El atacante debe haber comprometido previamente o poseer de manera legítima una cuenta con perfil de Administrador en el sistema afectado.
Interacción del Usuario (UI) Ninguna (None) UI:N El atacante puede detonar y ejecutar la vulnerabilidad por sí mismo, sin necesidad de engañar a otro usuario para que realice alguna acción.
Alcance (Scope - S) Sin Cambios (Unchanged) S:U El impacto del ataque queda estrictamente confinado a la aplicación afectada; no hay salto de privilegios al sistema operativo o a redes vecinas.
Impacto en Confidencialidad (C) Bajo (Low) C:L El atacante solo puede acceder de manera parcial a información no crítica o restringida; no hay exposición masiva de bases de datos.
Impacto en Integridad (I) Bajo (Low) I:L Las modificaciones a la información están restringidas y no alteran de manera destructiva el funcionamiento del negocio.
Impacto en Disponibilidad (A) Bajo (Low) A:L El sistema puede sufrir degradación menor en su velocidad de respuesta, pero no se interrumpe totalmente el servicio web.

4. Interpretación Técnica

A pesar de que el requisito de privilegios elevados (PR:H) actúa como un control de seguridad inicial muy robusto, la vulnerabilidad no debe descartarse ni perder relevancia en una auditoría de seguridad corporativa.

En un entorno de producción real, un atacante sofisticado que logre comprometer una cuenta administrativa a través de otros vectores previos (como fuerza bruta, inyección SQL o campañas avanzadas de phishing) utilizará inmediatamente este fallo de severidad media para realizar movimientos laterales o alterar información interna del sistema sin dejar rastro de interacción con el usuario. El perfil típico de amenaza para esta vulnerabilidad suele ser un insider (un empleado deshonesto) o un actor externo persistente (APT) en etapa avanzada. Dado que el impacto es Bajo en las tres dimensiones CIA (Confidencialidad, Integridad y Disponibilidad), la explotación no representa el colapso total de la infraestructura, pero sí genera una preocupante degradación en la confianza de los datos procesados por la aplicación.

Fórmula Base: La puntuación se calcula evaluando los coeficientes de las métricas de explotación e impacto. El requisito de altos privilegios (PR:H) actúa como un reductor de severidad, disminuyendo el puntaje base final a 4.3 sobre 10.0.

6. Conclusión

La evaluación sistemática mediante el modelo estándar CVSS v3.1 arroja un resultado numérico final de 4.3, situando de forma unánime esta vulnerabilidad en una categoría formal de Severidad Media (dentro del rango estándar de 4.0 a 6.9).

Desde una perspectiva práctica de gestión y priorización de riesgos corporativos, este hallazgo debe ser debidamente documentado en el inventario activo de seguridad de la organización. Se recomienda programar e implementar su remediación técnica (patching o parche de código) una vez que las vulnerabilidades catalogadas previamente como "Críticas" (9.0 - 10.0) e "Altas" (7.0 - 8.9) hayan sido mitigadas y cerradas con éxito. La principal medida preventiva y de control compensatorio para este caso específico radica en el robustecimiento de los mecanismos de autenticación y control de acceso (como implementar doble factor de autenticación MFA) y en el monitoreo estricto y en tiempo real de logs en las cuentas con privilegios administrativos para bloquear ataques de escalamiento lateral.
